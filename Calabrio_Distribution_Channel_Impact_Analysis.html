1. Field Overview & Purpose

Distribution_Channel__c is a restricted picklist field powered by the Distribution_Channel Global Value Set. It identifies whether a product, quote, opportunity, or service engagement is for Calabrio's Cloud, On-Premises, or Government Cloud deployment model.

Why It Matters

This field is a core business logic driver across the entire lead-to-revenue lifecycle. It controls:

AreaImpact
CPQ Pricing & BundlingDetermines which product options are available, which SKUs are required, and which pricing formulas apply. Drives 20+ references in CPQ custom scripts (QCP) and 13 error conditions.
Billing & TaxRoutes invoice lines and credit notes to the correct SOVOS tax API endpoints based on Cloud vs. On-Prem vs. Gov Cloud.
ProvisioningControls cloud tenant creation on Closed Won. Cloud/Gov channels trigger automatic tenant provisioning; On-Prem does not.
Service & EntitlementsSets SLA milestones: Cloud gets 120-minute first response, On-Prem gets 240-minute first response.
Billing HoldsOn-Prem renewal Maintenance/Support order products are placed on billing hold.
ReportingAvailable across 52 report types for segmentation by deployment model.

11. Risk & Migration Considerations

Change Impact Assessment

ScenarioRiskImpact
Add a new picklist value (e.g., "FedRAMP") HIGH Must update: Global Value Set, Constants.cls (add to Cloud or OnPrem list), all 9 flows with decision branches, CPQ summary variables, CPQ error conditions, CPQ custom scripts, and entitlement process milestone criteria. Without updates, the new value will bypass all automation.
Rename or deactivate a value HIGH All 9 flows, 4 Apex classes, 13 CPQ error conditions, and 3 CPQ summary variables contain hardcoded string comparisons. Renaming a value will break all downstream logic silently.
Remove the field from an object HIGH 37 permission sets, 52 report types, and multiple flows/Apex classes reference the field on specific objects. Removal will cause deploy failures and runtime errors.
Change from Global to local picklist MEDIUM Would require recreating values on each of 13 objects independently. Risk of value drift between objects. CPQ Configuration Attributes reference the global value set.
Migrate to a new org MEDIUM Must migrate: Global Value Set, field on all 13 objects, Constants.cls, all 9 flows, 4 Apex classes, CPQ configuration data, entitlement processes, 37 permission sets, and 52 report types. The field is a foundational dependency.

Known Gaps

GapDetailsRecommendation
Test coverage All 3 test classes only test with 'CAC - Calabrio Cloud'. No test coverage for On-Prem or Gov Cloud code paths. Add test methods for each Distribution Channel value, especially for needService() and tenant creation branches.
Hardcoded strings Flows and CPQ scripts use hardcoded string values rather than referencing the Constants class or custom labels. Consider migrating flow comparisons to use custom labels for easier maintenance.
Gov Cloud SLA gap Entitlement processes only define milestones for Cloud and On-Prem. Gov Cloud has no explicit SLA criteria. Verify whether Gov Cloud should follow Cloud SLAs or have its own milestones.
Deprecated perm set SalesOperationsDEPR still grants access to 11 objects for this field. Verify no active users are assigned, then remove.
