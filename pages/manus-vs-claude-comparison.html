1. What They Actually Are

Manus AI

Autonomous task-execution agent. You give it a goal -- "build me a competitive analysis report," "create a React prototype" -- and it plans all the steps, browses the web, writes code, generates files, and hands you a finished deliverable.

  • Runs in a cloud Ubuntu VM (not your machine)
  • You set the destination; it drives
  • Minimal human oversight by design
  • Launched March 2025 by Butterfly Effect (Singapore)
  • Acquired by Meta for ~$2B in December 2025
Claude (Web + Code CLI)

Claude Web is a conversational reasoning partner -- you drive the session, iterate on ideas, and decide next steps.

Claude Code CLI is an agentic developer assistant that runs on your local machine. It reads your codebase, edits files, runs terminal commands, interacts with git -- with you approving each action.

  • Your local filesystem, your repos, your tools
  • Semi-agentic: it plans and acts, you approve
  • Built by Anthropic
Key Insight Manus is a delegation tool (set a goal, walk away). Claude Code is a collaboration tool (work together on your codebase). They solve different problems.

2. What's Under the Hood

Here's the part most people miss: Manus is not its own AI model. It's an orchestration layer on top of existing foundation models.

Manus Architecture

  • Primary LLM: Claude 3.5 Sonnet v1 (at launch), testing Claude 3.7 for upgrades
  • Chinese-language model: Fine-tuned Alibaba Qwen
  • Action paradigm: "CodeAct" -- generates Python code as its primary execution mechanism
  • Environment: Cloud-hosted Ubuntu VM with shell access, headless browser (Selenium), Python/Node.js, full file system within the VM

Multi-Agent Loop

Planner Agent

Decomposes the goal into ordered steps

Execution Agent

Carries out steps using tools (shell, browser, code)

Monitor Agent

Evaluates progress and adjusts strategy

Translation When you use Manus, you're often paying Manus to call Claude for you, with automation wrapped around it. TechCrunch and The Decoder both confirmed this after leaked code analysis.

Claude Architecture

  • Model: Anthropic's proprietary Claude models (Opus, Sonnet, Haiku)
  • Claude Web: Runs on Anthropic's servers, conversational interface
  • Claude Code CLI: Runs locally in your terminal -- direct access to filesystem, git, shell commands, IDE integration
  • Context window: 400,000+ tokens (vs. Manus's ~200K)

4. Where Manus Wins

"Just Go Do It" Tasks

Research reports, competitive analyses, market reports where you don't want to manage each step. Give it a goal, get a deliverable.

Multi-Tool Workflows from Scratch

"Build me a prototype landing page" or "create an SEO audit from this URL" -- it chains web browsing + code + document generation autonomously.

Non-Technical Users

People who can't or don't want to set up dev environments. Manus gives them a computer that "just does things."

Strong Documented Use Cases

  • Research reports and competitive intelligence gathering
  • Candidate screening and interview scheduling (HR workflows)
  • SEO audits from a single URL submission
  • Real estate listing analysis and comparison documents
  • Front-end code generation (agencies report ~40% time savings on React components)
  • E-commerce product description writing (80% faster listing creation)
  • Full "idea to deployed prototype" workflows for non-developers

Benchmark Performance

On the GAIA benchmark (tests AI ability to reason, use tools, and automate real-world tasks), Manus scored 86.5%, outperforming OpenAI's GPT-4 and Deep Research on all three difficulty levels. This is a legitimate achievement in agentic task completion.

5. Where Claude Wins

Working on YOUR Codebase

Manus can't touch your local files, metadata, or git history. Claude Code lives in your terminal with full access to everything.

Precision & Control

You review every edit, every command. For production systems and client orgs, this step-by-step approval matters.

Security & Privacy

Your code never leaves your machine with Claude Code. Manus stores everything in Meta's cloud. Zero data retention is available with Claude.

Claude Code CLI Advantages

  • Local CLI tools: Run sf project deploy start, npm test, git commit directly from the same session
  • Longer context: 400K+ tokens means entire codebases in memory. Manus fragments on long tasks
  • Cost predictability: Flat monthly fee vs. Manus's opaque credit system
  • Proactive quality: Tests show Claude flags security issues, suggests better patterns, and explains trade-offs. Manus completes tasks but with less nuance
  • IDE integration: Works inside VS Code, Cursor, and other editors
  • Reasoning depth: Claude Opus/Sonnet are the same models Manus uses under the hood -- except you get them directly without the wrapper
Real-World Test Result When asked to write a WordPress plugin, Claude "flagged a security issue, suggested a more maintainable approach, followed coding standards without being asked, and explained decisions." Manus completed the same task but with less nuance and no proactive quality concerns raised. (Source: jock.pl AI agent comparison)

7. Criticisms & Concerns

Technical Reliability

  • Frequently gets stuck on CAPTCHAs and paywalled sites
  • Context window limits cause long tasks to fragment or fail
  • Documented bugs: empty ZIP file outputs, infinite browser refresh loops, task timeouts
  • High service load errors were common at launch due to viral demand

Transparency Concerns

  • Leaked code showed the system is essentially a wrapper around existing models (Claude, DeepSeek) with Selenium and web scraping
  • TechCrunch concluded it "probably isn't China's second DeepSeek moment" -- the hype outpaced the novelty
  • MIT Technology Review called it impressive but noted the marketing was ahead of the product

Privacy & Data Concerns

  • Originally collected IP addresses, keystrokes, and behavioral data stored on Chinese servers
  • Terms of service allowed generated content to be used for AI training
  • Independent monitoring found ~15MB of memory data uploaded hourly to a Shenzhen server
  • Tennessee banned Manus on government networks on launch day (March 6, 2025)
  • Post-Meta acquisition: Chinese ownership divested, but Meta's data practices are now the concern
  • Some enterprise customers left after the Meta acquisition

Marketing Tactics

  • Launched invite-only; invitation codes sold for up to 50,000 RMB (~$7K) on secondary markets
  • Criticized as manufactured scarcity ("hunger marketing")
Fair Balance Claude has its own limitations: it can hallucinate, it doesn't have real-time web browsing in the web app, and Claude Code CLI requires technical comfort with a terminal. No tool is perfect -- the question is which trade-offs matter for your work.

9. The Bottom Line

They're Not Really Competitors

Manus and Claude solve different problems. Manus is impressive for autonomous task delegation -- and it demos incredibly well ("watch it build a whole app from one sentence!"). But that's a different use case than working on an existing codebase with production systems.

  • Manus = "Here's my goal, go figure it out" (delegation)
  • Claude Code = "Let's work on this together in my environment" (collaboration)
  • Claude Web = "Help me think through this" (conversation)

When to Recommend Each

Choose Manus When...
  • Task is self-contained with a clear deliverable
  • Involves web research + document generation
  • You want to delegate and check back later
  • No sensitive/proprietary data involved
  • You're not a developer and need a "do it for me" tool
  • Building from-scratch prototypes, not modifying existing code
Choose Claude When...
  • Working on your existing codebase
  • Need CLI/terminal tool access (sf, git, npm, etc.)
  • Production systems where you need control per step
  • Data privacy matters (code stays local)
  • Complex reasoning and architecture decisions
  • Learning and understanding the "why" behind solutions
The Irony Manus uses Claude under the hood. When your buddy uses Manus, he's often paying Manus to call Claude for him -- with an autonomy wrapper and a cloud VM on top. You're going directly to the source, with more control and local access.
