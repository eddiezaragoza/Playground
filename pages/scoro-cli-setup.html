Setup Guide

Scoro + Claude
on your Mac

A complete walkthrough from a brand-new Mac to managing your Scoro projects with Claude Code CLI — no experience required.

🍎 macOS Claude Code CLI Scoro MCP
1
System
Install Homebrew
macOS's missing package manager — installs everything else
+
⏱ ~3 minutes
OPEN TERMINAL
Press ⌘ + Space, type Terminal, and hit Enter. This is your command-line interface — you'll use it for everything in this guide.
RUN INSTALLER
Paste this single command and press Enter:
Terminal
%/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"
⚠ Note
Your Mac will ask for your password. Type it (nothing will appear on screen — that's normal) and press Enter. It may also prompt you to install Xcode Command Line Tools — click Install if asked.
ADD TO PATH (Apple Silicon Macs)
If you have an M1/M2/M3/M4 Mac (most Macs from 2021 onward), run these two commands after installation finishes:
Terminal
# Add Homebrew to your shell profile %echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.zprofile %eval "$(/opt/homebrew/bin/brew shellenv)"
✓ Tip
Not sure which Mac you have? Click the Apple menu () → About This Mac. If "Chip" says M1/M2/M3/M4, you need these commands. If it says "Intel", skip them.
VERIFY
Confirm Homebrew installed correctly:
Terminal
%brew --version # Should output something like: Homebrew 4.x.x
2
Runtime
Install Node.js
Required to run Claude Code CLI
+
⏱ ~2 minutes
INSTALL WITH HOMEBREW
Node.js powers Claude Code under the hood. Install it via Homebrew:
Terminal
%brew install node
VERIFY
Check that both Node.js and npm are available:
Terminal
%node --version # Should output: v22.x.x or similar %npm --version # Should output: 10.x.x or similar
✓ Good to know
As long as you see version numbers — any version — you're good to go!
3
Core App
Install Claude Code
The CLI that lets you interact with Claude from your terminal
+
⏱ ~2 minutes
INSTALL GLOBALLY
Install Claude Code via npm so it's available anywhere on your system:
Terminal
%npm install -g @anthropic-ai/claude-code
SIGN IN
Launch Claude Code for the first time — it will open your browser to sign in with your Anthropic account:
Terminal
%claude # Your browser will open — sign in and return to the terminal
ℹ Info
If your browser doesn't open automatically, copy the URL shown in the terminal and paste it into your browser manually.
VERIFY
After signing in, confirm Claude Code is working:
Terminal
%claude --version # Should output a version number like: 1.x.x
4
API Key
Get Your Scoro API Credentials
Retrieve the credentials Claude needs to talk to Scoro
+
⏱ ~5 minutes
LOG IN TO SCORO
Open your Scoro instance in a browser and navigate to Settings → Integrations → API.
LOCATE YOUR CREDENTIALS
You will need these three values for the next step:
  • Company Account ID — your unique Scoro subdomain (e.g. yourcompany)
  • API Key — a long string used to authenticate requests
  • Site URL — your full Scoro URL (e.g. https://yourcompany.scoro.com)
⚠ Important
Keep your API key private. Never share it in public channels or commit it to version control.
5
Integration
Configure Scoro MCP Server
Connect Claude Code to your Scoro account via MCP
+
⏱ ~5 minutes
WHAT IS MCP?
MCP (Model Context Protocol) lets Claude Code talk directly to external services like Scoro. You'll register Scoro as an MCP server so Claude can read and manage your projects, tasks, contacts, and more.
ADD THE SCORO MCP SERVER
Run this command in your terminal to add the Scoro MCP server to Claude Code. Replace the placeholder values with your actual credentials from the previous step:
Terminal
%claude mcp add scoro \ -e SCORO_API_KEY=your_api_key_here \ -e SCORO_COMPANY_ID=your_company_id \ -e SCORO_SITE_URL=https://yourcompany.scoro.com \ -- npx -y @anthropic-ai/scoro-mcp-server
✓ Tip
You can edit this later by running claude mcp list to see your servers and claude mcp remove scoro to start over.
VERIFY THE CONNECTION
Confirm the MCP server was added successfully:
Terminal
%claude mcp list # You should see "scoro" listed with a ✓ status
6
Test Drive
Your First Scoro Query
Make sure everything works end to end
+
⏱ ~2 minutes
LAUNCH CLAUDE
Open Claude Code in any directory:
Terminal
%claude
ASK ABOUT YOUR PROJECTS
Once Claude is running, type a natural-language prompt to query Scoro:
Claude Code
> List all my active projects in Scoro
✓ What to expect
Claude will use the Scoro MCP server to call the Scoro API and return a list of your active projects, including names, statuses, and deadlines. If you see project data, everything is working!
TROUBLESHOOTING
If something doesn't look right:
  • Double-check your API key and company ID are correct
  • Run claude mcp list to confirm the server is registered
  • Restart Claude Code by typing /exit and then claude again
  • Make sure your Scoro account has API access enabled
⚡
You're all set.
Claude can now read and manage your Scoro data directly from the terminal. Here are some things to try:
"Show me overdue tasks across all projects"
"Summarize time entries for this week"
"Create a new task in Project X due Friday"
"List all contacts added this month"
