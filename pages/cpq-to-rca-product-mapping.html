Overview

This document maps every product-related object in Calabrio's CPQ implementation (CalabrioSITSandbox) to its corresponding object in the Revenue Cloud Advanced (RCA) implementation (VerintDevSandbox). It covers the full product data model -- catalog structure, bundle hierarchy, pricing, rules, configuration, and policies.

What are CPQ and RCA?

Salesforce CPQ (Configure, Price, Quote) is the legacy managed package (SBQQ__) for product configuration and quoting. Revenue Cloud Advanced (RCA) is the newer, platform-native replacement that uses standard Salesforce objects instead of managed package objects. The migration replaces CPQ's custom architecture with RCA's built-in product catalog, pricing engine, and lifecycle management.

Scope

This document covers product-related objects only -- the catalog, pricing, bundles, rules, and configuration that define what can be sold and how it's priced. It does not cover quote/order transaction objects (SBQQ__Quote__c, SBQQ__QuoteLine__c, etc.) or downstream billing/revenue recognition.

Catalog & Categories

RCA introduces a formal product catalog structure that CPQ does not have. In CPQ, products are organized implicitly through bundle features. In RCA, products live in a catalog with navigational categories.

CPQ (SIT) -- No Catalog

CPQ has no catalog concept. Products are organized by:

  • Product Family -- picklist field on Product2 (WFM, CAS, AQM, ONE, BCR, BA)
  • Bundle Features -- SBQQ__ProductFeature__c groups options into tabs within a bundle configurator

There is no browsable catalog or category hierarchy. Products are found via search or bundle configuration.

RCA (Verint) -- Product Catalog

RCA uses a formal catalog with navigational categories:

  • ProductCatalog -- "Default Catalog" (single catalog for all products)
  • ProductCategory -- Navigational categories within the catalog
  • ProductCategoryProduct -- Junction linking products to categories

Migration Pattern

CPQ Product Features map conceptually to RCA ProductCategories. The feature name (Software, Services, Custom Services) becomes the category name. However, categories are org-wide and navigational, while features are bundle-specific. In RCA, a product belongs to a category regardless of which bundle it's in.

Bundles & Hierarchy

Both CPQ and RCA support product bundles (a parent product with configurable child options). The mechanics differ significantly.

CPQ Bundle Architecture

  • Bundle indicator: Product2.SBQQ__ConfigurationType__c = "Allowed" or "Required"
  • Parent-child link: SBQQ__ProductOption__c (junction with quantities, pricing flags, feature assignment)
  • Grouping: SBQQ__ProductFeature__c (tabs in the configurator UI)
  • Constraints: SBQQ__OptionConstraint__c (dependency/exclusion rules between options)

Key ProductOption Fields

  • SBQQ__OptionalSKU__c (child Product2)
  • SBQQ__ConfiguredSKU__c (parent bundle)
  • SBQQ__Feature__c (which tab/group)
  • SBQQ__Quantity__c, SBQQ__Required__c
  • SBQQ__Bundled__c (price included?)
  • SBQQ__Type__c (always "Related Product")

RCA Bundle Architecture

  • Bundle indicator: Product2.Type = "Bundle" (immutable after creation)
  • Parent-child link: ProductRelatedComponent (with relationship type, quantities, pricing flags)
  • Grouping: ProductComponentGroup (analogous to features)
  • Relationship type: ProductRelationshipType (defines the nature of the parent-child link)

Pricing

Pricing is the most complex area of the CPQ-to-RCA migration. CPQ uses a layered system of block prices, price rules, discount schedules, and a JavaScript QCP. RCA replaces all of this with PriceAdjustmentSchedules, PricingRecipes, and Expression Sets.

Block Pricing Pattern

5,726 block price records define quantity-based tiers. Example: 1-100 agents = $8,800, 101-500 = $7,500, etc. This is the dominant pricing mechanism for PS SKUs.

Discount Schedules

7 schedules (CAS, ONE, QM, WFM, CR, BA, Analytics Enterprise), all "Slab" type with percent-based tiers by agent count (0-250 = 0%, 251-500 = 7-8%, up to 10000+ = 42-60%).

QCP (JavaScript)

12 versions of the Quote Calculator Plugin. Handles overage pricing, currency exchange, product type summarization, quantity multipliers, and custom field calculations. All logic in JavaScript.

Price Adjustment Schedules

3 schedules exist (Standard, Attribute Based, Bundle Based) but with 0 tiers configured. These will hold the migrated block prices and discount tiers.

Pricing Recipe

1 recipe with 10 table mappings configured. Recipes orchestrate the pricing calculation flow, replacing the QCP's JavaScript with declarative Expression Sets.

Expression Sets (replaces QCP)

RCA uses Expression Sets (ExpressionSetDefinition) instead of JavaScript. All QCP logic (overage pricing, currency exchange, aggregations) must be rebuilt declaratively.

Rules & Logic

CPQ product rules control what can be selected together, what's automatically added, and validation. RCA replaces this with qualifications, disqualifications, and configuration flows.

Selection rules (31) auto-add/remove products based on conditions. Example: "Add SP-CASANA support SKU when analytics software is selected and distribution channel is On-Prem Perpetual."

Key condition fields tested: Distribution_Channel__c, Perpetual__c, SBQQ__ProductCode__c, Product_Type__c

RCA Rules (Verint)

All rule objects exist in the schema but have zero records. The entire rule engine needs to be rebuilt.

Key difference: CPQ rules are condition-action pairs evaluated by the QLE engine. RCA uses qualifications (eligibility criteria) and Salesforce Flows for configuration logic.
